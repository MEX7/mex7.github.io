Kinect来操作浏览器

Jul 12, 2014

#背景说明

打算使用Kinect来控制浏览器里面的一些动作，比如拖动和点击
此处暂时只考虑 chrome 使用 Xbox360 的硬件设备
目前安装过很多版本的驱动,最后确定使用 OpenNI 2.2 + Nite 2.2 + Kinect SDK 1.8
完成上述安装之后， Kinect 可以正常使用但是在浏览器中无法检测到设备已经连接，所以暂时无法成功，不知道问题原因。

吐槽

关键点卡在Kinect如何能成功被激活，现在看到2010年MIT大大写的depthjs也有视频演示证明别人确实是成功了，但是我很疑惑啊，明明是比较有前景的市场，为何一直没人来做。微软自己也不爱惜自家的IE。别人家的chrome的kinect插件装上去没反应，我猜想微软内部提供给IE一个连接kinect的插件也会比较有卖点哇。
现在就是下图这副模样，推进不了。 现在图跪了
不过说实话也不一定必须仰仗depthjs，同时也尝试过KinectInBrowser这个东西，但是装完之后毫无反应，我也没深究了。
和实验室做kinect的小伙伴探讨过这个问题，也没什么进展，目前实验室对kinect的研究也仅仅只在模拟键盘输入的部分，但具体是什么我也不懂啊。

求大神指点我啊
